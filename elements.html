HTML Elements

The following four column table presents the HTML element information on (41) commonly used tags that I have learned thus far:
Tag Description Block, Inline or N/A Container or Empty
<!-- --> HTML comments are invisible on a browser unless the source code is viewed. They are used to provide instructions and credit n/a container
<!DOCTYPE> Mandatory first declaration in an HTML document that specifies rendering instructions for your browser n/a empty
<a> Defines a hyperlink, which is used to link from one page to another or to another location within a page. The all important href attribute frequently used within the tag indicates the link's destination inline container
<article> Provides a semantic label to independent, self-contained content that should make sense on its own and could be placed or reused separately from the rest of the site. Common examples are blog posts or comments block container
<aside> Used to describe, summarize or identify content that is tangentially related to the primary content of a page, or the content surrounding the aside. Frequently used for ads or pull quotes block container
<body> Defines and contains all the content of an HTML document, e.g. text, links, images and tables like this one block container
<br> Inserts a single line break, but is not intended for separating paragraphs inline empty
<dd> Description tag is used to describe a term or name in a description list and is always used together with the <dl> and <dt> tags. It can contain paragraphs, images, links, etc. block container
<div> A generic container element used as a last resort when no other semantically rich elements apply group related content, to apply styles with CSS and or perform JavaScript tasks block container
<dl> Stands for description list and contains the the <dt> and <dd> tags that define and describe terms/names block container
<dt> Stands for description term - this tag defines a term/name in a description list block container
<em> A semantically rich, logical tag that renders emphasized text both visually and for assistive devices inline container
<fieldset> Used to group related elements in a form and draws a box around the contained elements. If a caption is needed, the <legend> tag is helful block container
<figcaption> Used within the <figure> element to define a caption within that container and can be either the first or last child block container
<figure> Used to mark up "self-contained content" like illustrations, photos, code listings, etc. Although related to the main content of the page, this element's position is independent and if taken out of the page, it wouldn't affect the document flow or meaning block container
<footer> Defines the footer of the document or section and can be used more than once in a document. This tag usually contains:
  • authorship information
  • copyright information
  • contact information
  • back to top links
 block container
<form> Used to create an HTML form for user input in conjunction with a number of other form elements, e.g. <input>, <textarea>, <button> etc. block container
<h1> to <h6> Used to define HTML headings and provide semantic structure to the HTML document via the heading number block container
<head> The head element occurs after the <!DOCTYPE> and <html> tags and contains all of the meta data for the page including but not limited to the <title>, <meta>, and stylesheet <link> tags n/a container
<header> Groups introductory content and is frequently used to contain a set of navigational links. The header element typically contains one or more heading elements, logos or icons, authorship information etc. There can be several <header> within an HTML document block container
<html> Representing the root of the document and acting as the container for all other HTML elements - besides the <!DOCTYPE> tag - this tag tells the browser that the document is in fact an HTML document n/a container
<iframe> Specifices an inline frame that embeds another document, such as content from a third-party webpage like Google Maps or Youtube, within an HTML document. This MDN tutorial includes additional info and cautions on using iframes n/a container
<img /> Defines an image in an HTML page. The first required attribute src="path/imageName.png" links to the actual location of the image - this could be either a path to the folder where the image file is located or an absolute url. The alt="description of image" attribute increases accessibility and will also display if the src path breaks. inline empty
<input> Used within the <input> element to specify an input field where the user can enter data. The input field itself can vary greatly in its look and function depending on the value passed to the type attribute inline container
<label> Provides a literal lable for other elements such as: <button>, <input>, <meter>, <output>, <textarea>. Although it does not render anything special, its a usability improvement that toggles the control if the user clicks or taps within the text contained by the <label> element. In order to bind the label with a related control, the for attribute should = the id attribute of the control inline container
<li> Defines a list item and is used in ordered lists <ol> and unordered lists <ul> block container
<link /> Defines and establishes a link between the current HTML document and an external resource. Primary use is to link to external stylesheets via the attribute value rel="stylesheet" n/a empty
<meta> Metadata is information about data and as such, the <meta> element - always contained within the <head> provides metadata about the HTML document that isn't displayed on the page via its attribute name/value pairs. This info often tells browsers how to display content or reload a page and frequently specifies a page description, keywords, author of the page etc. used by search engines or other web services n/a empty
<nav> Defines a set of navigational links and is intendend only for major blocks of navigation links although there may be other links on the page. This tag is new as of HTML5 block container
<ol> Defines an ordered list that can be numerical or alphabetical depending on the value of the 'type' attribute. Its items are definied with the <li> tag. HTML5 introduced the 'reversed' attribute for ordered lists block container
<p> Defines a paragraph and it is generally recommended to wrap any text that is not intended for another semantically rich purpose in <p> tag. Browsers add a default margin before and after each <p> element, which can be modified or removed with CSS block container
<script> Used to define client-side scripts e.g. with JavaScript. This element either contains scripting statments or directs to an external script file via a src= attribute. If src is used, the script must be empty. Generally, <script> is placed at the end of the document before the closing </body> tag either depending contents container or empty with src="scripts.js"
<section> Literally defining sections within a document, this new HTML5 tag provides wrapped content with a semantically rich, logical label. The theme of each section should be identified by including a heading element <h1> - <h6> as a child block container
<strong> A logical tag that not only makes the wrapped text visually bold, but it also gives it the semantic meaning of "important text". Styling through CSS is still preferable, but this tag can be helpful when considering how assistive devices read text inline container
<style> Contains style information via CSS rules for a document or part of the HTML document. Athough it can be placed in the <body> element, best practice is to place styles in the <head>. For ideal organization, would be to place CSS in an external stylesheet and apply them via <link> elements, although there are cases when internal styling is necessary n/a container
<table> Defines a grouping of elements that make up an HTML table such as this one. The contained elements must include one or more <tr>, <th>, and <td>, but more complex tables may include sectioning elements are frequently included such as <caption>, <tfoot> and <tbody> block container
<td> Defines a standard cell withn an HTML table row - see <tr>. Unlike <th> header cells, these standard cells contain data and by default are regular case and left-aligned. n/a container
<th> Defines a header cell within an HTML table row <tr>. Header information, such as table column names are wrapped by this tag. By default <th> elements are bold and centered n/a container
<title> Required in all HTML documents, it literally defines the title of the document. Located in the <head>, the <title> element:
  • provides a title in the browser toolbar and tab
  • Is the label for the page when added to favorites
  • Is the title that is listed for the page search-engine results
 n/a container
<tr> The mark up for a row in an HTML table. This element contains one or more <th> and or <td> elements as its children. n/a container
<ul> Generates an unordered (bulleted) list and contains <li> elements as the list items block container
